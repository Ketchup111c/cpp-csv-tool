# CSV 结构化解析工具


## 一、选题说明与项目功能介绍

### 1.1 选题背景

本项目为AI协同开发综合考核项目，基于纯C++标准库从零自研CSV结构化解析工具，不依赖任何第三方解析库。
原生C++标准库仅提供基础文件读写能力，无法适配现实业务中复杂CSV场景：字段内嵌逗号、双引号转义、跨行文本、空行脏数据、行列不匹配、超大体积文件等。
项目采用多轮迭代模式逐步开发，全程依托AI辅助完成编码、排错、测试、文档撰写。

### 1.2 项目功能介绍

- **标准 CSV 解析**：正确处理字段内的逗号、双引号（`""` 转义）与换行。
- **表头映射**：读取首行作为表头，建立「列名 → 列索引」映射，支持按列名取值。
- **字段清洗**：自动去除字段首尾空白；过滤全空行与含空字段的脏数据行。
- **数值统计**：按指定列名遍历有效数据行，计算总和与平均值（严格数值校验，非数字字段报错）。
- **关键词筛选**：按关键词（任意列包含即匹配，区分大小写）筛选数据行。
- **安全导出**：`CsvWriter` 在字段含逗号/双引号/换行/首尾空白时自动加引号包裹，
  内部双引号转义为 `""`，并保留原表头。
- **健壮的异常处理（统一异常体系）**：
  - `CsvIoException`（IO 错误）：文件打开/写入/路径非法。
  - `CsvFormatException`（格式错误）：数值解析失败、列名不存在。
  - `CsvColumnMismatchException`（行列不匹配）：数据行列数与表头不一致（空白行除外）。
  - `CsvEncodingException`（编码异常）：字段含非法 UTF-8 字节。
  - `CsvArgumentException`（参数错误）：命令行参数缺失或未知。
- **大文件友好**：流式逐行读取与写出，单行内存占用恒定，避免内存溢出。
- **编码与格式校验**：自动剥离 UTF-8 BOM，逐字段校验 UTF-8 合法性。


## 二、提示词与 JSONL 数据集生成方法

### 2.1 提示词编写方法
- 先借助智能体将整体开发任务拆解为独立子任务，再基于每个子任务针对性编写对应提示词：

- **文档提示**：给定固定大纲，强制 Markdown 规范输出；
- **统一要求**：使用代码块包裹，需求分层清晰。
  
### 2.2 JSONL 数据集生成方法

1. 打开 Kilo Code 聊天窗口，输入导出命令 /export，将完整对话记录导出为 Markdown 文件；

## 三、 编译与运行

### 3.1 依赖

- C++11 及以上编译器（g++ / clang++ / MSVC 均可）。
- 仅使用 C++ 标准库，无需第三方依赖。

### 3.2 使用 g++ 直接编译

```bash
g++ -std=c++11 -I include src/csv_reader.cpp src/main.cpp -o csv_tool.exe -static
.\csv_tool.exe -f example.csv -col id -filter e -out result.csv
```

### 3.3 命令行参数

| 参数 | 说明 |
| --- | --- |
| `-f <path>` | 必填，原始 CSV 文件路径 |
| `-col <name>` | 可选，指定用于求和/均值的数值列名 |
| `-filter <kw>` | 可选，按关键词筛选（任意列包含即匹配，区分大小写） |
| `-out <path>` | 可选，将结果（筛选后或清洗后）导出为 CSV |
| `-h` | 打印帮助文档 |

### 3.4 命令示例

```bash
# 读取并清洗，对 id 列做统计
./csv_tool -f example.csv -col id

# 完整流程：统计 + 关键词筛选 + 导出
./csv_tool -f example.csv -col id -filter e -out result.csv

# 仅筛选并导出（不做统计）
./csv_tool -f example.csv -filter Alice -out alice.csv

# 查看帮助
./csv_tool -h
```

示例输出（`-col id -filter e -out result.csv`）：

```
表头（列名 -> 列索引）:
  [0] id
  [1] name
  [2] note

有效并匹配行数: 4
被过滤的无效/不匹配行数: 4

数值统计（列 "id"）:
  行数 = 4
  总和 = 13.00
  平均值 = 3.2500

已导出结果至: result.csv
```

异常会被统一捕获并按类别提示，例如列数不匹配时：

```
[CSV 行列不匹配] 第 8 行列数不匹配：期望 3 列，实际 4 列
```

## 四、 开发问题与解决方案

- **CSV 引号 / 跨行分割错乱**：使用状态机解析，引号内不分割，""转译为单个引号；
大文件内存溢出：采用流式逐行读取，不缓存全部数据；
- **空行 / 行列不匹配导致崩溃**：前置校验脏行，统计无效行数，异常捕获跳过错误行；
- **找不到头文件**：编译增加-I include；
Windows 无法运行 exe：执行时添加.\前缀；
- **中文控制台乱码**：终端切换 UTF-8 编码，源码保存为 UTF-8；
- **参数报错**：必须用-f指定文件，不能直接传文件名；
- **Git 推送连接重置**：切换网络或配置 Git 代理。

## 五、 测试方法说明

边界与异常用例测试位于 `tests/test_edge.cpp`，覆盖：

1. 打开不存在文件 → `IO 错误`
2. 空文件 → `readHeader` 返回 `false`
3. 数据行列数与表头不一致 → `行列不匹配`
4. 字段含非法 UTF-8 字节 → `编码异常`
5. 数值列含非数字内容 → `格式错误`
6. 文件带 UTF-8 BOM → 首列表头名被正确剥离
7. 含逗号/双引号/换行的转义字段 → 解析正确且列数一致
8. 空行 / 空字段 → 被清洗过滤（有效 + 无效计数正确）
9. 写入到非法路径 → `IO 错误`
10. 大文件流式读取 10 万行 → 逐行计数全部成功，不占满内存
